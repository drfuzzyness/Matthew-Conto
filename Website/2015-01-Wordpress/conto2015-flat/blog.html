Ideas for a better Digital Scrapbook
February 4th, 2015 - from a series on Notetaking Apps

The key to this note taking app is seamless workflow.

AwesomeBar extended

The Firefox AwesomeBar is pretty self explanatory. When I had a stint using Google Chrome, the URL bar always bothered me as it favored search results and exact URLs from my History over the URL that I wanted to go to. trello would be auto completed to something like https://trello.com/b/zttawhOT/ instead of https://trello.com. It wasn’t fun for me. With that in mind, fiddling with the default behavior of the AwesomeBar is dangerous.

Keywords

My first thought would be to require a keyword to initiate the Notational Velocity style search/new note interface. It solves the issue of separating web requests like trello from note requests like ideas for band names. As easy as the keyword solution would be to implement, it feels lackluster. It fails my dream of a seamless gateway between the web and the mind.

The Most Obvious

And then it hit me. Duh, this’ll just work right at the start! The AwesomeBar’s engineers, in deciding what a user might use the bar for included bookmark support! If you start to search for trello, you’ll be confronted with any bookmarks you have that partially match the string trello! The AwesomeBar handles what results to display first, but the result is visible as a bookmark!

The next step is to implement new note creation. The keyword solution remains the easiest to handle indifference Notational Velocity has towards search and note creation. There are several solutions to handle this creation.

  • Searches that don’t exactly match a note show a link to create new note
    • Extra action to create note
  • Press a modifier shift+enter to create a new note with the title from the URL bar
    • Separates notes from the URL and could slow down the user creating a note as they have to memorize a shortcut.
    • Not a bad solution, but not instantly understandable.

This will require more thought.

A way to make bookmarks useful
January 26th, 2015 - from a series on Notetaking Apps

I’ve don’t have a good how I’ll actually make this, but here goes.

Bookmarks as Data

Firefox Bookmarks have a URL, a hierarchy, and a notes section. I can’t imagine that anyone actually uses the notes section, but what if someone did? What if, you stuck an entire note taking app inside of a bookmark. Just simple Markdown to format, with a specialized URL to handle viewing the bookmark as a note.

It’s been done

QuickFox is, to be frank, a pretty lame app. It looks like it was pulled out of the 2005 Windows XP school of application design with the content taking the full width of the screen and relying on a window hovering above your Firefox tabs. Even if you try to load the addon as a tab, the addon relies on its own internal tabs. But even with that said, it’s a truly brilliant idea. Your notes are stored as bookmarks, thereby making sync a simple and unobtrusive experience as your browser handles it in the background through whatever service you want.

I just want to take that one step further and turn it into full blown bookmark/scrapbooking system along the lines of Evernote.

The Mailbox App of Note-taking

Note-taking isn’t that glamorous. For the longest time I’ve relied on Evernote to handle formatting and sync and it works pretty well. It provides more metadata than I could ask for with support for authors, tags, even location. What it doesn’t do is offer a simple service. In fact, it’s a fucking mess.

If you ‘clip’ a website into Evernote, you’re stuck with the silly formatting and bastardized data structure that the site used. My gripe isn’t with the algorithms they’re using to harvest the content, but the presentation of the content. Personally, when I take notes, I want my result to be clean and to-the-point. Basically, I want some nice clean Markdown I can turn into HTML if it needs to be styled. I don’t want minute control over the fonts, the line-height, the colors for each individual note. I want one style and appropriate formatting.

Browsers and Notational Velocity

Notational Velocity is is brilliant. nvALT is, so far, my favorite brand of it with awesome Markdown support, custom font display support, and Simplenote. The key to Notational Velocity is its reliance on the search bar, much like a certain other piece of critical software, the browser URL bar.

What if the browser URL bar could not only be the gateway to the web, but the gateway to your memories.

Pretty lofty, right? Every startup needs a tagline like that. So imagine your reading list merged with your notes. A certain workflow that encourages documentation of thoughts.
